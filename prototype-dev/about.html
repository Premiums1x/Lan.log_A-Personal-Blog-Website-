关于这个博客

「lancer.log」开始于 2020 年的春天。那段时间我刚开始独力维护一个线上服务，每天回家后脑子里全是别人说过的话、写过的代码、提过的需求，几乎没有一句是自己的。我开始写，最初的目的是想让语言回到自己身上——把一天里碎片化的感觉，重新组织成一段连贯的句子，像把一堆 raw log 聚合成一份可读的 report。

写着写着，它变成了一个更长期的东西：一个可以被未来的自己回来 git blame 的地方。所以这里的内容不会追热点，也很少写"现在"——更多是过一段时间之后，那件事还值不值得再被想起。

关于写作方式

我坚持一次只写一篇，写完放几天再回头看。每篇文章平均要改四到六稿，最长的一篇改了三个月——很像一次漫长的 code review，只是 reviewer 只有我自己。这并不是什么了不起的事，只是一个适合我节奏的速度；再快，我就会开始写自己没想清楚的东西。

这个博客只有两种状态：published 和 draft。没有"热门文章"、"推荐榜单"这样的设置。所有文章按时间倒序陈列，就像 git log --oneline，你按自己的心情选着读就好。

一些不写在订阅号里的事

这里没有算法，没有推送，没有"在看"。如果你愿意来读，那就来读；过了几个月没来，我也会安静地在这里继续写下去。这是一份理解：好的阅读关系，和好的代码协作关系一样，都是慢的、有可能失联的、但不需要每天都确认的。
