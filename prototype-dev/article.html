关于"够用就好"这件事，
我用了七年才真正学会

极简不是扔东西。它是在每一次拥有之前先确认：这件东西是否值得它将要占据的空间、注意力， 以及之后每一次的轻微愧疚。一篇写给七年前那个自己的笔记。

许 xu · published2026-07-05 · read8 min · words1,248 · commita1f3c2d

关于"够用就好"这件事，我用了七年才真正学会。说"七年"并不夸张——那是从我开始第一次尝试简化生活，到现在能够稳定地、不再用力地保持它的全部时间。中间绕过的弯路，比这条结论本身长得多。

早些年，我理解的极简是减法：把柜子里多余东西扔掉，把日程上的事删掉，把社交里消耗我的人清理掉。这套做法短期内很有效，柜子空了，日历也清爽，整个人好像轻了许多。但过不了多久，那些被腾出来的空间，会被更精致、更"值得拥有"的新东西重新填满——只是这次它们换了一层更好看的外壳。

这就像一个只删代码不重构的工程师：bug 暂时没了，但代码库的复杂度还在那儿，下一次需求一来，问题照样长回来。

问题不在东西，在"够"这个词

后来我才意识到，真正难的不是减少，是定义"够"。东西少到什么程度算够，时间留给多少事算够，留多少空间给自己算够——这些问题因人而异，且随年龄变化。但只要你没有一个属于自己的、可以长期持有的"够"，减法就永远是临时的。

我们这一代人，大多是"多就是好"的默认设置里长大的。commit 要多，star 要多，技能要多，去过的地方要多。哪怕在做减法的时候，脑子里也还是这个逻辑：

text
// 我已经删了 60% 的依赖，
// 那再删 20%，是不是更"高级"？
// —— 看，连极简也变成了又一场关于"更多"的竞赛。

一个我用了三年的检验问题

后来我给自己定了一个很小的检验问题，在买任何一件非消耗品前问一遍：

这件东西将占据多少注意力？——不是空间，是注意力。

—— 写给自己的一条 lint 规则

注意力和空间不一样。一双很少穿的鞋占的衣柜空间很小，但它每次开门都会提醒你一次"我应该多穿它"，然后是轻微的愧疚，再然后是麻木。一年里这种小小的愧疚多了，会在不知不觉里消耗掉一些心神。

!
核心区别：当开始用"注意力"去衡量，而不是用"体积"，很多原本觉得"还好"的东西，忽然变得很重。这和 evaluate code smell 一样——行数少不代表成本低。

三个慢慢收敛下来的原则

  • 先空三个月。想买的，先记在清单里。三个月后还想买，再认真考虑。绝大多数想买的，在清单上停三天就消失了。这条规则帮我砍掉了 90% 的"工具理性"消费。
  • 一件换一件。同类物品只保留一件，旧的用坏之前不买新的。这条让我的工具数量稳定在了一个常数上，类似 dependency 里的 peerDependencies——只在确实需要时引入。
  • 明确"够"的数字。外套五件，马克杯两只，笔记本同时只用一本。给"够"一个具体的数，避免无止境的微调，就像配置项一旦定下就不再 review PR 去改它。

最难的一步：不再解释

真正学会了之后，最难的一步不是维持，是不再向别人解释。极简一旦说出口，就容易被归类成"一种风格"或"一种追求"，而它本来只是我想对自己过下去的一种方式。当我停止解释，那些重复的、用来证明自己生活是"对"的力气，才真正卸下来了。

七年下来，留下的东西不算少，但每一件我都知道它为什么在。够用就好，不是一种克制，而是一种不需要再克制。这时候，"够"才真的发生了。
